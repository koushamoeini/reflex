## فعالیت‌های انجام شده توسط کوشا معینی

### Step 1: ساختار اولیه و پیکربندی
در این مرحله، اسکلت اصلی پروتکل Reflex در هسته Xray بنا شد:
- ایجاد پکیج `reflex` در مسیر `xray-core/proxy/reflex`.
- تعریف ساختارهای داده برای تنظیمات ورودی (Inbound) و خروجی (Outbound).
- ثبت پروتکل در لیست توزیع Xray در فایل `main/distro/all/all.go`.

### Step 2: فرآیند Handshake و احراز هویت
در این مرحله، منطق ایجاد ارتباط امن و پنهان پیاده‌سازی شد:
- پیاده‌سازی Handshake ضمنی (Implicit Handshake) برای ناشناس ماندن ترافیک.
- استفاده از تبادل کلید `X25519`.
- پیاده‌سازی سیستم احراز هویت بر اساس `UUID`.
- استفاده از `HKDF` برای استخراج کلیدهای نشست.

## فعالیت‌های انجام شده توسط آرمان آریان‌پور

### Step 3: Traffic Morphing (پنهان‌سازی ترافیک)
در این مرحله، سیستم پنهان‌سازی ترافیک برای جلوگیری از شناسایی پروتکل پیاده‌سازی شد:
- پیاده‌سازی الگوریتم Traffic Morphing برای تبدیل بسته‌های داده به ترافیک HTTP معمولی.
- افزودن هدرهای HTTP واقع‌گرایانه به بسته‌ها.
- استفاده از Padding تصادفی برای پنهان کردن اندازه واقعی داده‌ها.
- پیاده‌سازی تست‌های جامع برای تایید صحت عملکرد morphing.

### Step 4: تست‌های جامع و اعتبارسنجی
در این مرحله، مجموعه کاملی از تست‌ها برای اطمینان از کیفیت کد نوشته شد:
- پیاده‌سازی Unit Tests برای اجزای مختلف پروتکل.
- نوشتن Edge Case Tests برای بررسی رفتار در شرایط خاص.
- ایجاد Benchmark Tests برای سنجش عملکرد.
- پیاده‌سازی Fuzz Tests برای یافتن باگ‌های احتمالی.
- نوشتن Example Tests برای مستندسازی نحوه استفاده از پروتکل.
